## Engineering Objective
This software acts as an independent compliance registry. Instead of training machine learning models to maximize standalone accuracy metrics, this framework parses classification outputs against statutory legal boundaries and non-parametric statistical tests to verify whether automated pipelines display adverse impact toward protected demographic characteristics.

## Architecture & Methodological Pillars

### 1. Production Model Pipeline Integration
The application builds a production classification system by loading the real-world **UCI German Credit Dataset**, isolating feature spaces, vectorizing categorical fields via structural encoding, and fitting an active operational machine learning model (**Random Forest Classifier**). The auditor parses the real output predictions ($y_{\text{pred}}$) generated by this classifier.

### 2. Statistical Compliance Framework
Rather than relying on single arbitrary scalar metrics, the engine validates systemic risk via three mathematical indicators:
* **Demographic Parity Ratio (The Four-Fifths Rule):** Calculates the relative selection variance between group vectors ($\mu_{\text{protected}} / \mu_{\text{reference}}$). It triggers a structural alert if the ratio drops below the regulatory statutory boundary of `0.80`.
* **Chi-Square ($\chi^2$) Test of Independence:** Constructs a dynamic $2 \times 2$ contingency matrix mapping the absolute distribution frequencies of group selection attributes. It calculates the definitive $p$-value to test the null hypothesis ($H_0$). If $p < 0.05$, the system rejects the null hypothesis and flags the variance as statistically significant systemic bias.
* **False Rejection Rate (FRR) Disparity:** Measures the algorithmic performance equity by quantifying the absolute difference in false-negative errors between protected populations, capturing underlying algorithmic mistreatment.
